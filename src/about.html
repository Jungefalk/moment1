Arbetsprocessen

I mitt projekt har jag skapat en package.json fil med hjälp av npm init. I filen har jag lagt in ett script som heter build. Parcel använder build för att minimera och optimera filerna. Då tas onödiga mellanrum, kommentarer och överflödig kod bort.

I package.json-filen har jag lagt till "browserslist" enligt Parcels dokumentation för att definera vilka webbläsare sidan ska vara kompatibel med. Parcel transpilerar modern JavaScrip-kod för att säkerställa att webbplatsen fungerar även i äldre webbläsare.

Jag har sedan skapat ett GitHub-repository för mitt projekt med två branches, en main branch och en development branch.

Jag har kopplat min main branch till webbhosten Netlify. När main branch uppdateras upptäcker Netlify ändringarna och kör scriptet npm run build. Då uppdateras webbplatsen automatiskt med optimerade filer.

I min development branch kan jag arbeta med projektet utan att det påverkar den publicerade webbplatsen. När jag sedan är nöjd så mergar jag ihop development branch med main branch och sparar för att uppdatera den publicerade webbplatsen.
